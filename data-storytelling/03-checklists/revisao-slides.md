# Checklist: Revisão de Slides

> Use **antes** de apresentar. Se falhar em qualquer item, revise.

## Estrutura

- [ ] Abertura: título, autores, contexto (quem, o quê, por quê)
- [ ] Ato 1: problema e importância estão claros
- [ ] Ato 2: 1 descoberta principal por slide
- [ ] Ato 3: conclusão + call-to-action
- [ ] Apêndice: slides extras para perguntas difíceis

## Conteúdo

- [ ] Mensagem principal aparece pelo menos 3 vezes (início, meio, fim)
- [ ] Cada slide tem uma única mensagem principal
- [ ] Título de cada slide conta uma história (não é apenas "Resultados")
- [ ] Números e dados têm fontes (quando relevante)
- [ ] Nada é mostrado "por via das dúvidas"

## Design

- [ ] Consistência visual (cores, fontes, estilos) em todos os slides
- [ ] Fonte mínima 18pt para corpo (30pt+ para títulos)
- [ ] Contraste suficiente (nada de texto claro em fundo claro)
- [ ] Sem logos grandes ou elementos decorativos repetitivos
- [ ] Espaço em branco presente (slides não estão "cheios")

## Gráficos

- [ ] Todos os gráficos passaram no checklist de revisão de gráficos
- [ ] Tipo de gráfico adequado aos dados
- [ ] Cores com propósito (não arco-íris)
- [ ] Legível quando projetado (teste em tamanho real)

## Preparação

- [ ] Apresentação testada em voz alta (pelo menos 1x)
- [ ] Timing cronometrado (dentro do limite)
- [ ] 3 perguntas difíceis identificadas com respostas preparadas
- [ ] Apêndice pronto com slides de suporte
- [ ] Backup (pendrive, nuvem, e-mail)
- [ ] Equipamento testado (projetor, clicker, laptop)

## Durante a Apresentação

- [ ] Slides são seus lembretes, não seu teleprompter
- [ ] Pausas antes dos dados importantes
- [ ] Transições verbais entre slides ("Isso nos leva a...")
- [ ] Contato visual com a plateia (não com os slides)
- [ ] Se algo der errado: respire, continue, adapte
